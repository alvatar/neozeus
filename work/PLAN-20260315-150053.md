# Summary of feature

Implement a custom retained HUD subsystem rendered with Vello, using explicit message passing + dispatcher, persistent module layout, draggable module shells, and the first module `AgentList`.

Target end-state:

- HUD is **not** built on egui windows or Bevy UI widgets
- HUD module shells are draggable and persist their last rect
- HUD modules communicate through a typed dispatcher
- module-local state is retained in Rust structs, not in render output
- Vello scene is pure derived output
- module `1` is `AgentList`
- `AgentList` lists terminals as agents (`agent-1`, `agent-2`, ...)
- agent labels are changeable later through a message-driven rename path
- clicking an agent row focuses its terminal and triggers a presentation policy that hides/minimizes the others without destroying them or affecting their runtime state
- the current EVA demo visuals are removed; only reusable Vello HUD architecture remains
- the old egui toolbar is removed and replaced by a Vello-rendered `DebugToolbar`

Must not regress:

- terminal rendering / input
- multi-terminal independence at the domain/runtime level
- reactive redraw behavior
- startup robustness / persistence failures must not abort app startup
- no new dependencies

- keep only reusable Vello draw helpers and HUD scheduling API in the new HUD subsystem
- `src/hud/mod.rs` becomes the narrow API consumed by `scene.rs`

### Invariants
- removal of EVA demo visuals must not remove Vello integration itself
- removal of egui toolbar must not remove essential controls; those controls move into `DebugToolbar`

### Traps
- do **not** carry over old demo-specific scene generation into the new HUD renderer
- do **not** keep temporary egui ownership alive “just for now”; it will immediately distort the design

## Phase 2 — Introduce retained HUD state + module registry

### Goal
Create one source of truth for shell/layout/module state.

### Core types

```rust
#[derive(Clone, Copy, Debug, PartialEq, Eq, Hash, PartialOrd, Ord)]
pub(crate) enum HudModuleId {
    DebugToolbar,
    AgentList,
}

impl HudModuleId {
    pub(crate) const fn number(self) -> u8 {
        match self {
            Self::DebugToolbar => 0,
            Self::AgentList => 1,
        }
    }

    pub(crate) const fn title(self) -> &'static str {
        match self {
            Self::DebugToolbar => "Debug Toolbar",
            Self::AgentList => "Agent List",
        }
    }
}

#[derive(Clone, Copy, Debug, Default)]
pub(crate) struct HudRect {
    pub(crate) x: f32,
    pub(crate) y: f32,
    pub(crate) w: f32,
    pub(crate) h: f32,
}

#[derive(Clone, Debug)]
pub(crate) struct HudModuleShell {
    pub(crate) enabled: bool,
    pub(crate) z: i32,
    pub(crate) target_rect: HudRect,
    pub(crate) current_rect: HudRect,
    pub(crate) target_alpha: f32,
    pub(crate) current_alpha: f32,
}

#[derive(Clone, Debug, Default)]
pub(crate) struct AgentListState {
    pub(crate) scroll_offset: f32,
}

#[derive(Clone, Debug, Default)]
pub(crate) struct DebugToolbarState;

#[derive(Clone, Debug)]
pub(crate) enum HudModuleModel {
    DebugToolbar(DebugToolbarState),
    AgentList(AgentListState),
}

#[derive(Clone, Debug)]
pub(crate) struct HudModuleInstance {
    pub(crate) shell: HudModuleShell,
    pub(crate) model: HudModuleModel,
}

#[derive(Resource, Default)]
pub(crate) struct HudState {
    pub(crate) modules: BTreeMap<HudModuleId, HudModuleInstance>,
    pub(crate) z_order: Vec<HudModuleId>,
    pub(crate) drag: Option<HudDragState>,
    pub(crate) dirty_layout: bool,
}
```

### Agent metadata source
Add an explicit metadata resource now so names are not hardcoded forever:

```rust
#[derive(Resource, Default)]
pub(crate) struct AgentDirectory {
    pub(crate) labels: BTreeMap<TerminalId, String>,
}
```

Default label resolution:
- if `AgentDirectory` has a label: use it
- else derive `agent-{n}` from terminal order

### Registry
Static definitions are separate from runtime state:

```rust
pub(crate) struct HudModuleDefinition {
    pub(crate) id: HudModuleId,
    pub(crate) default_enabled: bool,
    pub(crate) default_rect: HudRect,
}
```

### Invariants
- shell/layout state is generic and module-agnostic
- module content state is local to the module
- domain truth remains outside HUD modules
- human-readable agent labels come from one centralized source

### Traps
- do **not** put Vello scene objects into `HudState`
- do **not** make `AgentList` the pattern for shell state; shell must stay generic
- do **not** scatter default `agent-{n}` formatting across render/input code

## Phase 3 — Add typed dispatcher with targeted and broadcast events

### Goal
Let modules emit commands and receive events through an explicit dispatcher.

### Recommended message split

```rust
pub(crate) enum HudCommand {
    FocusTerminal(TerminalId),
    HideAllButTerminal(TerminalId),
    RenameAgent { terminal_id: TerminalId, label: String },
    SetModuleEnabled { id: HudModuleId, enabled: bool },
    ToggleModule(HudModuleId),
}

pub(crate) enum HudEvent {
    TerminalFocused(TerminalId),
    TerminalPresentationIsolated(TerminalId),
    TerminalSpawned(TerminalId),
    TerminalClosed(TerminalId),
    AgentRenamed { terminal_id: TerminalId, label: String },
}

pub(crate) enum HudRecipients {
    One(HudModuleId),
    Some(Vec<HudModuleId>),
    All,
}

pub(crate) struct HudEnvelope<T> {
    pub(crate) recipients: HudRecipients,
    pub(crate) payload: T,
}

#[derive(Resource, Default)]
pub(crate) struct HudDispatcher {
    pub(crate) commands: Vec<HudCommand>,
    pub(crate) events: Vec<HudEnvelope<HudEvent>>,
}
```

### Delivery model
- input/content logic pushes `HudCommand`
- app/domain system drains commands and mutates real state
- that system emits `HudEvent` envelopes
- module update system dispatches events to interested modules

### Why this split
- commands are requests
- events are facts
- recipients support one / many / all modules
- broadcasting is explicit, not implicit magic

### Invariants
- modules do not mutate `TerminalManager` or presentation state directly
- broadcasts go through one place
- targeted delivery is supported from day 1

### Traps
- do **not** use events for local hover/drag state
- do **not** turn every internal UI mutation into a dispatched message
- do **not** let `AgentList` itself hide other terminals directly; it emits commands only

## Phase 4 — Add HUD persistence for shell layout

### Goal
Persist module shell state without adding dependencies.

### Persisted model
Persist only shell data first:

```rust
pub(crate) struct PersistedHudModuleState {
    pub(crate) enabled: bool,
    pub(crate) rect: HudRect,
}

pub(crate) struct PersistedHudState {
    pub(crate) modules: BTreeMap<HudModuleId, PersistedHudModuleState>,
}
```

### File path
Use XDG config path with explicit fallback:
- default: `${XDG_CONFIG_HOME}/neozeus/hud-layout.v1`
- fallback: `${HOME}/.config/neozeus/hud-layout.v1`

### Format
Use a tiny custom line format; avoid `serde`/RON dependency churn.

Example:

```text
version 1
DebugToolbar enabled=1 x=24 y=24 w=920 h=56
AgentList enabled=1 x=24 y=96 w=300 h=420
```

### Write policy
- load once at startup
- mark `HudState.dirty_layout = true` on drag / resize / enable change
- save on a debounce timer or on a small cooldown system

### Invariants
- corrupted/missing layout file must not break startup
- saved coordinates are logical HUD-space coordinates

### Traps
- do **not** persist transient state like drag-in-progress or hover row
- do **not** block the render/input path with heavy file I/O every frame

## Phase 5 — Add HUD input + drag controller

### Goal
Hit-test module shells and route content interaction.

### Input split

#### Shell handles
- titlebar hit testing
- dragging
- raising module to front
- module enable/disable by number later

#### Module handles
- clicks inside content area
- scroll inside content area

### Core state

```rust
pub(crate) struct HudDragState {
    pub(crate) module_id: HudModuleId,
    pub(crate) grab_offset: Vec2,
}
```

### Systems
- collect cursor position from primary window
- hit test top-most module under pointer
- if titlebar press:
  - start drag
  - raise module z-order
- while dragging:
  - update `target_rect.x/y`
  - mark layout dirty
- on pointer release:
  - clear drag

### Content routing
For `AgentList`:
- content hit test returns row under pointer
- click row emits:
  - `HudCommand::FocusTerminal(id)`
  - `HudCommand::HideAllButTerminal(id)`
- wheel scroll mutates `AgentListState.scroll_offset`

### Invariants
- shell drag must not be owned by module content
- hit testing uses retained rects, not Vello scene inspection

### Traps
- shell vs content rectangles must be separate or clicks will misroute
- dragging must request redraw in reactive mode

## Phase 6 — Add HUD animation state

### Goal
Animate shell rect/alpha independently of rendering.

### Execution
Each frame:
- `current_rect -> target_rect`
- `current_alpha -> target_alpha`

Simple exponential or lerp-based smoothing is enough.

```rust
current.x += (target.x - current.x) * blend;
current.y += (target.y - current.y) * blend;
```

### Invariants
- current state is animated output
- target state is authoritative layout target

### Traps
- do **not** write persistence from animated current rect; save target rect
- redraw scheduler must consider active HUD animation

- label resolution:

### Click behavior
Clicking a row emits:


### Highlighting

### Minimal local state

```rust

    pub(crate) scroll_offset: f32,
```

### Traps
- do **not** duplicate active terminal state into the module model
- do **not** bake `agent-{n}` formatting into multiple files

## Phase 10 — Wire command application + event dispatch + terminal presentation policy

### Goal
Make module commands actually change app state and produce events.

### Systems

#### `apply_hud_commands`
- drain `HudDispatcher.commands`
- apply to `TerminalManager` / `HudState` / terminal presentation state / `AgentDirectory`
- emit events

Example:

```rust
match command {
    HudCommand::FocusTerminal(id) => {
        terminal_manager.focus_terminal(id);
        dispatcher.events.push(HudEnvelope {
            recipients: HudRecipients::All,
            payload: HudEvent::TerminalFocused(id),
        });
    }
    HudCommand::HideAllButTerminal(id) => {
        terminal_visibility_policy.isolate(id);
        dispatcher.events.push(HudEnvelope {
            recipients: HudRecipients::All,
            payload: HudEvent::TerminalPresentationIsolated(id),
        });
    }
    HudCommand::RenameAgent { terminal_id, label } => {
        agent_directory.labels.insert(terminal_id, label.clone());
        dispatcher.events.push(HudEnvelope {
            recipients: HudRecipients::All,
            payload: HudEvent::AgentRenamed { terminal_id, label },
        });
    }
    HudCommand::ToggleModule(id) => { ... }
    HudCommand::SetModuleEnabled { id, enabled } => { ... }
}
```

#### `dispatch_hud_events`
- drain event queue
- deliver to one / many / all module models

`AgentList` may ignore most events initially beyond refresh-worthy ones.

### Presentation policy for “hide others”
Do **not** overload terminal focus with hiding logic. Add a separate presentation policy resource, e.g.:

```rust
#[derive(Resource, Default)]
pub(crate) enum TerminalVisibilityPolicy {
    #[default]
    ShowAll,
    Isolate(TerminalId),
}
```

Presentation systems read this resource and hide/minimize/de-emphasize non-selected terminals.

Critical semantic rule:
- hidden terminals still exist
- hidden terminals keep their runtime/backend state
- hiding is presentation state only, analogous to minimizing a window
- showing them again must not require respawning/recreating the terminal

### Invariants
- module → domain mutations go only through commands
- events are facts after mutation, not requests
- focus and presentation isolation stay conceptually separate

### Traps
- avoid mutating `HudState.modules` while iterating if delivery code borrows broadly
- keep command application before animation/render in schedule order
- do **not** model “hide others” by destroying terminals or dropping presentation state

## Phase 11 — Integrate redraw demand + validation

### Goal
Keep HUD reactive and visible without breaking existing redraw logic.

### Redraw integration
Extend redraw demand to include:
- active HUD drag
- HUD layout dirty with pending save/animation
- any module shell animation in progress
- future content animations

### Schedule order
Add HUD sets explicitly:

```rust
HudInput
-> HudCommands
-> HudEvents
-> HudAnimation
-> HudRender
-> Redraw
```

### Validation
Required checks:
- `cargo test`
- `cargo clippy --all-targets -- -D warnings`
- `cargo fmt --check`
- runtime smoke
- manual checks:
  - `DebugToolbar` visible
  - `AgentList` visible
  - drag works
  - position persists across restart
  - clicking row focuses terminal
  - clicking row hides others via presentation policy
  - typing still repaints terminal under reactive mode

### Invariants
- terminal pipeline correctness unchanged
- `AgentList` drag/persistence must not block terminal interactivity

### Traps
- reactive mode will hide redraw bugs immediately if HUD input/animation does not request redraw
- persistence path errors must be logged and ignored, not fatal

# Final implementation rules

- no egui windows for modules
- no Bevy UI widgets for module layout
- no trait-object plugin system yet; use explicit enum-based modules
- no render-state-as-truth; Vello scene is disposable output
- no direct module mutation of terminal/domain state; use commands
- use `events`, not `notifications`
- no persistence of transient interaction state
- no new dependencies
